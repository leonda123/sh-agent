## 四、页码连续性检查（强制步骤，必须执行）

**【重要】这是必须执行的检查步骤，不可跳过！**

### 4.1 提取所有页码标识
从 pages.json 中提取所有 `page_label` 字段，形成完整的页码序列。

### 4.2 检查页码连续性
必须逐一检查以下问题：

**【关键】首先识别页码格式！**

页码格式可能是以下三种之一：
1. **纯阿拉伯数字**：1, 2, 3, 4, 5...
2. **纯罗马数字**：I, II, III, IV, V...
3. **罗马数字+阿拉伯数字组合**：I, II, III, IV, V, VI, VII, 8, 9, 10...（目录用罗马数字，正文用阿拉伯数字）

**【重要】罗马数字+阿拉伯数字组合是正常情况！**

如 I, II, III, IV, V, VI, VII, 8, 9, 10... 这种目录用罗马数字、正文用阿拉伯数字的情况是**正常**的，不是跳号！
检查时应该分别检查罗马数字部分和阿拉伯数字部分是否各自连续。

1. **页码格式识别**：
   - 先判断页码格式是纯阿拉伯数字、纯罗马数字、还是罗马数字+阿拉伯数字组合
   - 输出页码格式：[纯阿拉伯数字/纯罗马数字/罗马数字+阿拉伯数字组合]

2. **起始页检查**：
   - 正文页码是否从 1 开始？
   - 如果是罗马数字+阿拉伯数字组合，正文从阿拉伯数字1开始即可
   - 如果正文从其他页码开始（如从第5页开始），说明前面可能有封面、目录等非正文页

3. **页码跳号检查**：
   - **在同一页码格式内部**检查是否连续
   - 纯阿拉伯数字：检查 1, 2, 3, 4... 是否连续
   - 纯罗马数字：检查 I, II, III, IV... 是否连续
   - 罗马数字+阿拉伯数字组合：分别检查罗马数字部分和阿拉伯数字部分是否各自连续
   - 例如：1, 2, 4, 5（缺少3）就是跳号

4. **页码重复检查**：
   - 检查是否存在重复的页码
   - 例如：1, 2, 3, 3, 4（重复3）就是重复

5. **非附录页码重置检查（关键）**：
   - **这是最常见的错误！**
   - 检查页码序列中是否存在突然从大数字变成小数字的情况
   - 例如：37, 38, 1, 2, 3... 这种情况是**严重错误**
   - **判定规则**：如果页码序列中出现"..., 37, 38, 1, ..."或"..., 20, 21, 22, 1, ..."这种模式，且该位置附近没有明确标注"附录"字样，则判定为**页码重置错误**
   - **必须检查**：在页码重置位置附近的内容中搜索"附录"关键词，如果没有找到，则判定为错误
